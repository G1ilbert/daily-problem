# Day 002 — Parse Request Line + Routing

## โจทย์วันนี้
ทำให้ server รับ request แล้ว redirect ไปยัง URL หลายที่

## แก้ยังไง
นำ HTTP Request Message ในส่วน request line ดึง url ที่ต้องการออกมาตรวจสอบ
ใช้ strstr ค้นหา GET แล้วเลื่อนตำแหน่งไปชี้ตัวใหม่ หาช่อง space แล้วเปลี่ยนเป็น \0


## ผลทดสอบ
```bash
$ curl -v http://localhost:8080/abc123
< HTTP/1.1 302 Found
< Location: https://youtube.com

$ curl -v http://localhost:8080/xyz789
< HTTP/1.1 302 Found
< Location: https://github.com

$ curl -v http://localhost:8080/unknown
< HTTP/1.1 404 Not Found
```
